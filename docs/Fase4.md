# Fase 4 - Metodología de trabajo

## Metodología elegida

Para el desarrollo del proyecto DUOFUT he decidido utilizar una **metodología mixta basada en Scrum y Kanban**.

He elegido este modelo porque me permite combinar lo mejor de ambos: por un lado, la organización por fases y objetivos de Scrum, y por otro, la visualización clara del trabajo que ofrece Kanban. Esto me facilita organizarme mejor y llevar un control más claro del proyecto.

- **Scrum** lo usamos para dividir el trabajo en pequeños bloques y tener objetivos concretos.
- **Kanban** lo usamos con el Trello para ver en todo momento en qué estado está cada tarea.

Además, esta combinación se adapta bastante bien al tipo de proyecto que estamos realizando.

---

## Organización del trabajo (Scrum)

El proyecto lo he organizado en **sprints cortos**, de aproximadamente una semana.

En cada sprint tengo en cuenta:
- Los objetivos que quiero cumplir
- Las tareas necesarias para conseguirlos  
- El reparto del trabajo  

Al finalizar cada sprint reviso lo que he hecho y ajusto la planificación si hace falta.

---

## Uso de Kanban (Trello)

Para el seguimiento de tareas estoy utilizando Trello, aplicando el método Kanban.

He creado un tablero con las siguientes columnas:

- Pendiente  
- En progreso  
- En revisión  
- Completado  

Cada tarea es una tarjeta que voy moviendo entre columnas según voy avanzando.

Esto me permite ver de forma rápida qué tareas tengo hechas, cuáles están en proceso y cuáles faltan, lo que facilita mucho la organización.

---

## Organización de reuniones

Para coordinar el trabajo utilizo Microsoft Teams.

He planteado diferentes tipos de reuniones:

- **Reunión inicial:** Para organizar el proyecto  
- **Reuniones de seguimiento:** Para ver el progreso y detectar problemas  
- **Reunión final:** Para revisar el resultado  

Intento que las reuniones sean breves y centradas en:
- Qué he hecho
- Qué voy a hacer
- Qué problemas han surgido

---

## Seguimiento del trabajo

El seguimiento del proyecto lo realizo utilizando varias herramientas:

- **Trello:** para controlar el estado de las tareas  
- **GitHub:** para guardar y gestionar los cambios del proyecto  
- **Microsoft Teams:** para la comunicación  

Además, voy actualizando la documentación en Markdown con MkDocs para reflejar el progreso del proyecto.

---

## Justificación de la metodología

He decidido utilizar una metodología mixta Scrum + Kanban porque:

- Me permite organizar el trabajo de forma clara  
- Me facilita ver el progreso en todo momento  
- Mejora la organización y la planificación  
- Se adapta bien a un proyecto pequeño como este  

En general, esta forma de trabajar ayuda a mantener el proyecto ordenado, avanzar de forma progresiva y tener un mejor control de todo el proceso.
